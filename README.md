# GLM 技术方案智能生成 · 演示 POC

基于智谱 GLM 大模型与智谱线上知识库的技术方案智能生成演示系统：上传招标/需求文件，
经「需求解析 → 大纲规划 → 分段检索生成 → 整合校验」四阶段生成完整技术方案并导出 Word；
支持客户上传历史技术方案到知识库（自动向量化），生成时逐章检索参考，全程流式可见、可干预。

## 四阶段机制

```
招标文件 ──▶ ① 需求解析          GLM 深度解析：技术参数/工期/资质/评分标准/隐含风险
              │
              ▼
          ② 大纲规划            知识库检索相似历史方案 → 生成大纲 → 人工编辑确认
              │
              ▼
          ③ 分段检索生成         逐节「检索参考 → 约束组装 → 流式生成」+ 全文术语统一
              │
              ▼
          ④ 整合校验            完整性/一致性/关键条款校验 → 问题清单 → 导出 Word
```

## 目录结构

- `app/` 单体 FastAPI 应用（`zhipu/` 智谱 API 封装、`pipeline/` 四阶段流水线、
  `prompts/` 可覆盖的阶段提示词、`routes/` 接口、`static/` 免构建前端）
- `samples/` 演示素材：3 份历史技术方案 + 1 份招标文件
- `scripts/` `make_samples.py` 素材生成、`smoke.py` 真实接口冒烟
- `docs/` 设计 spec 与实施计划

## 演示脚本

1. **知识库管理**页：展示已有 3 份历史方案（状态「成功」）；可现场上传一份新文档，
   观察「处理中 → 成功」
2. **方案生成**页上传 `samples/招标文件-某开发区智慧园区平台项目.docx` → 开始解析，
   展示流式解析过程与五板块结果，重点讲「隐含风险预警」（评分倾向、本地化服务等）
3. 下一步进入**大纲规划**：展示检索到的历史方案参考片段（有据可依）→ 现场改一个章节
   标题 / 删一节 → 确认大纲
4. **分段生成**：逐节流式输出，左侧导航打勾推进，右侧术语表增长，进度条到 100%
5. **整合校验**：跑校验看问题清单（可点章节号跳回正文）→ 导出 Word 用 WPS/Word 打开
6. 需要时点右上角「提示词设置」展示客户可自定义各阶段提示词

## 已知实测要点（智谱知识库 API）

- 上传文档必须显式传 `knowledge_type=1`（文章知识），默认动态解析会误判「文档损坏」
- `embedding_stat` 显示滞后：成功以「能检索出片段」为准，失败以 `failInfo` 为准
